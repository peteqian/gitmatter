This section is for **developers and integrators** building on top of gitcounsel.
Unlike the rest of the docs, it is technical: it documents the exact tool surface,
HTTP endpoints, and authentication flows rather than how to use the product day to day.

gitcounsel has two programmatic surfaces, both backed by the same access checks and
git-style audit spine:

- An **MCP server** at `/api/mcp`, exposing a catalog of tools to AI agents (Claude,
  ChatGPT, and other MCP clients) and to the in-app assistant.
- A **REST API** under `/api`, used by the web app and available to integrators.
