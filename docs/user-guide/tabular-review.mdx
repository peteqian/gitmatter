---
title: Tabular review
description: Ask one question across a stack of documents and collect every answer in a table.
---

A **tabular review** lets you ask the same question (or several questions) across many
documents at once, and lines up all the answers in a neat table. Think of a
spreadsheet: one row for each document, one column for each question.

For example, say you have 50 contracts to get through. With a tabular review you can,
for every single one, pull out the governing law, the length of the contract term,
and the renewal date — and read all the results side by side instead of opening each
file one at a time.

## How it works

1. **Create a review** and add the documents you want it to read — for example, your
   stack of vendor contracts.
2. **Add columns.** Each column is one question you want answered for every document
   — for example, "Governing law", "Contract term", or "Renewal date". A column is
   just the heading for that question.
3. gitcounsel **reads each document and fills in the table.** Each box where a row and
   a column meet (one document, one question) gets its own answer. These boxes are
   often called cells, like in a spreadsheet.
4. Every answer comes with a **citation** — that is, the exact spot in the original
   document where the answer was found. So you are never asked to take an answer on
   trust; you can go and check it.

When the system "reads the document and fills in the answer", it is going through the
text, finding the part that responds to your question, and writing the answer into the
table for you.

## Checking and re-running

You can click into any cell to see its answer and the citation behind it — the passage
in the source document the answer came from.

If you change a column's wording, or you simply want a fresh reading, you can
**re-run** a cell. gitcounsel will read the document again and fill in a new answer.
Each reading is kept on record, so you can see how a cell's answer changed over time,
and who — a person or the assistant — set off each reading.

## Exporting

Once the table looks right, you can export it to a spreadsheet file (**CSV or XLSX**).
From there you can open it in a program like Excel, or pass it along to colleagues.

## Why the citations matter

Because every answer points straight back to its source, and every change is kept on
record, a tabular review is never a mystery box. A reviewer can rely on an answer
because they can see exactly where it came from in the document and confirm it in
seconds.
