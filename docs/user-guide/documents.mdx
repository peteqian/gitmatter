Documents are at the centre of gitcounsel. You upload a contract, the system reads it
for you, and from there you can read it, edit it, and let the assistant suggest
changes — with a full record of everything that happens kept along the way.

## Uploading a document

You can add a document from inside a matter, or from the **Documents** screen.
gitcounsel accepts the file types you would expect, such as PDF and Word, up to a set
size limit.

Once you upload a file, the system reads it and turns it into clean text that you can
read and search. This work happens quietly in the background — you do not have to wait
and watch. The document's status will change to show you when it is ready.

If something goes wrong while reading a file, you will see that too, and you can ask
the system to try again from the document itself.

## Reading and reviewing

Open a document to read it on screen. The page is built for comfortable reading: the
text is given plenty of width and clear, easy type, so working through a long
agreement does not feel cramped.

## Redline (suggested edits shown as tracked changes)

A **redline** is a set of suggested edits to the document. They are shown as tracked
changes — the same idea as the "Track Changes" feature in Microsoft Word, where you
can see exactly what someone proposes to add or remove before anything is final.

Here is how it works:

1. The assistant, or one of your colleagues, proposes an edit. Each edit is a specific
   change to the wording, and it comes with a short reason explaining why.
2. You see each proposed change marked clearly in the text.
3. You decide, one change at a time, whether to **accept** it or **reject** it.

Nothing is ever changed behind your back. Every suggested edit, and every accept or
reject you make, is saved together with the name of the person who did it. You can
always see the original wording and trace how the document reached the state it is in
now.

## Versions

A document keeps a history of its versions. When a document is replaced with a newer
file, the new content is added on top — the older version is not thrown away. That
means you can always go back and look at how the document read before.

## Exporting

You can download a document at any time. The assistant can also create a brand-new
Word (`.docx`) file for you as part of a workflow — handy when you want a fresh draft
that you can send out to someone.
