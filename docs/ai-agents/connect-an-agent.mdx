---
title: Connect an agent
description: Connect Claude, ChatGPT, or another AI assistant to gitmatter.
---

Connecting an AI assistant to gitmatter takes a few minutes and you only do it once.
There are two ways to sign in:

- **The easy way (recommended): a secure login.** You sign in to gitmatter and tap
  "approve" — the same way you connect any other service to an app. There is no code
  to copy and paste. This is called OAuth. It is how Claude Desktop and ChatGPT
  connect.
- **A secret code (a token).** gitmatter can create a secret code, like a password,
  that you give to your AI tool. This is mainly used by technical command-line tools.

Either way, the assistant acts as **your** gitmatter account, and everything it does
is recorded under the assistant's name in your history.

## The easy way: sign in and approve

This path needs no code. You add gitmatter inside your AI tool, sign in, and approve
access. "Connector" below just means the feature in your AI tool for adding outside
services.

### ChatGPT (Developer Mode, paid plans)

In ChatGPT, go to **Settings → Connectors** and add a custom connector. For the
address, use gitmatter's connection address: `https://<host>/api/mcp` (replace
`<host>` with your gitmatter web address). ChatGPT then takes you to a gitmatter
sign-in and approval page. Approve it, and you are connected. There is nothing to copy.

### Claude Desktop / web (custom connector)

In Claude, add a connector using the same address: `https://<host>/api/mcp`. Claude
takes you through the same sign-in and approval. Approve it, and you are connected.

<Callout type="warn">
  A note mostly for people running their own gitmatter server or testing on their own computer: the
  sign-in-and-approve method needs gitmatter to be reachable at a real, secure web address (one that
  starts with `https://`). If you are testing on your own machine, you can use a tool like
  `cloudflared` or `ngrok` to create a temporary secure address that points at your local gitmatter,
  then use that address in place of `http://localhost:4280`. If you are using a normal hosted
  gitmatter, this does not affect you.
</Callout>

## For technical users

The rest of this page is for developers using command-line tools. If you are a lawyer
connecting Claude or ChatGPT, you can skip it — the easy way above is all you need.

Most of these tools can use either the secure sign-in (OAuth) or a secret code (a
token). The plugin/OAuth path below is easiest; the token path is handy for a
self-hosted or local instance.

### Create a token

Only needed for the token path below. Go to **Settings → Connect** and create a
token. The full token is shown **once** — copy it then. If you lose it, revoke it
and create a new one.

### Claude Code (CLI)

Easiest — install the plugin. This repo is also a Claude Code plugin marketplace, so
it wires up the connection and signs you in over the secure login (no token to copy):

```bash
claude plugin marketplace add Git-Matter/gitmatter
claude plugin install gitmatter@gitmatter
claude plugin enable gitmatter
```

Approve the gitmatter sign-in the first time the agent uses a tool.

Or point Claude Code straight at a gitmatter address with a static token (useful for a
self-hosted or local instance):

```bash
claude mcp add --transport http gitmatter http://localhost:4280/api/mcp \
  --header "Authorization: Bearer <token>"
```

### Codex CLI

Easiest — add gitmatter and sign in over the secure login (no token to copy):

```bash
codex mcp add gitmatter --url https://gitmatter.com/api/mcp
codex mcp login gitmatter
```

`codex mcp login` opens a browser approval page. gitmatter supports dynamic client
registration, so there is no client secret to configure.

Or use a static token against a self-hosted or local instance — add this to
`~/.codex/config.toml`:

```toml
[mcp_servers.gitmatter]
url = "http://localhost:4280/api/mcp"
bearer_token_env_var = "GITMATTER_TOKEN"
```

### How the sign-in (OAuth) side works

This section is for developers who want the technical details of the secure login.
gitmatter is an OAuth 2.1 resource + authorization server. It serves discovery at
`/.well-known/oauth-protected-resource` and `/.well-known/oauth-authorization-server`,
runs the authorization-code flow with PKCE (S256), and supports both Client ID
Metadata Documents (ChatGPT's preferred path) and dynamic client registration. Access
tokens are bound to the gitmatter server address (the token only works here) and the
login step reuses your existing gitmatter session. For the full token API, see
[Authentication](/api-reference/authentication).
