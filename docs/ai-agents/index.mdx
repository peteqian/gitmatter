gitcounsel is built to be run by an AI assistant you already use. You connect a
tool like **Claude Desktop**, **Claude web/CLI**, or **ChatGPT** to gitcounsel one
time. After that, you ask it in plain language — _"review these NDAs"_ — and it does
the work for you inside gitcounsel. You stay in the AI tool you like; gitcounsel does
the legal work behind the scenes and keeps a full record of it.

Here is the important part: **anything the AI assistant does is recorded just like
work done by a person.** It is logged under the assistant's name, in the same history
as everyone else's work. Connecting an assistant does not create a hidden back door
that skips this record.

## How it connects (in plain words)

AI tools and other software talk to each other using a shared standard called **MCP**
(Model Context Protocol) — think of it as a common language that lets your AI tool
plug into outside services. gitcounsel speaks this language. To connect, you simply
point your AI tool at gitcounsel's connection and sign in. The next page walks you
through it.
